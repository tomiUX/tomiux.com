01 // CONTEXT

The Setup

Starbucks has scale. Philz has intention. This evaluation asks whether more resources produce better usability — or whether Philz's smaller, more deliberate design clears the bar on the heuristics that matter most.

Method

Applied Jacob Nielsen's 10 Usability Heuristics to both mobile apps. Each heuristic was evaluated independently, identifying interaction design strengths and weaknesses. Screenshots captured in live app sessions, February 2026.

Scope

Evaluation focused on core ordering flows: browsing, customization, cart, checkout, and error states. Loyalty program screens and account settings were assessed where they intersected with primary tasks.

Starbucks App

Global coffee chain. Mature mobile platform with millions of daily active users. App handles ordering, loyalty rewards, payment, and store locating. High design investment and frequent iteration.

Philz Coffee App

West Coast specialty coffee chain. Smaller footprint, fewer resources. The app handles ordering and rewards with a conversational, personality-forward design approach that distinguishes it from category norms.

H01

Visibility of System Status

☕ Starbucks

Makes effective use of loading indicators by confirming when an item has been added to the cart. Could improve visibility of real-time order status post-placement and offer users the option to cancel during delays.

Cart confirmation toast appears immediately after adding an item.

🟠 Philz Coffee

Provides excellent, immediate feedback. Button text changes dynamically in real-time as the user interacts, keeping users informed at every step without requiring them to search for confirmation.

"I'll Try It!" — before tapping

"YAY!" — button confirms action

H02

Match Between System and the Real World

☕ Starbucks

Uses familiar, plain language ("Choose a Store") and universal icons throughout. The ordering workflow mirrors real-world counter experience, making navigation intuitive for new and returning users alike.

🟠 Philz Coffee

Takes real-world language further with a conversational customization interface. Instead of dropdowns and toggles, users build their drink in a sentence — the same way they'd order in person. The result genuinely feels like talking to a barista.

The order builds as a sentence, mirroring how you'd speak to a barista.

H03

User Control and Freedom

☕ Starbucks

Offers strong pre-order control: detailed order summary before confirmation, multiple edit controls on cart items, and clear personalization options. The gap is post-placement — once an order is submitted, there's no cancellation path.

Four item-level controls (edit, favorite, add, delete) before checkout.

🟠 Philz Coffee

Cart items are editable and customizations can be adjusted freely. The pain point is reordering with minor modifications — users must restart from the main menu rather than duplicating and editing an existing order, adding unnecessary friction for repeat customers.

H04

Consistency and Standards

☕ Starbucks

Maintains a consistent design language across colors, fonts, and icons, and follows platform navigation standards reliably. Minor inconsistencies appear in non-core sections — particularly help articles — but these don't impact primary task flows.

🟠 Philz Coffee

Visual consistency is strong overall, with a clear navigation bar and uniform language. One notable miss: the checkout page uses a black button where the established brand palette calls for brown or orange — a jarring inconsistency at the most critical moment in the flow.

H05

Error Prevention

☕ Starbucks

Sold-out items are visually flagged in-menu with red iconography and plain language, preventing users from selecting unavailable items before clicking through. Step-by-step ordering guidance reduces error opportunities throughout the flow.

Red X badges prevent selection of unavailable items in the carousel.

Item detail reinforces unavailability before a user can add to cart.

🟠 Philz Coffee

Uses constraints and real-time validation to prevent mistakes at the input level. Invalid payment entries are immediately flagged with a red icon, guiding users to correct the issue before they can proceed — no form submission required to surface the error.

Inline validation flags the invalid card number in real-time.

H06

Recognition Rather than Recall

☕ Starbucks

Reduces memory load by keeping features like Favorites persistently visible. The gap: support tools are buried in navigation menus, forcing users to recall the specific path rather than recognizing them at a glance.

🟠 Philz Coffee

Strong recognition-based design throughout. The cart icon always shows an item count, and a "Recent Order" shortcut surfaces on the home screen — letting frequent customers reorder without navigating the full menu.

"Recent Order" card with one-tap RE-ORDER on the home screen.

H07

Flexibility and Efficiency of Use

☕ Starbucks

Accessible and intuitive for all user levels, but underserves power users. A one-tap reorder feature would reduce friction for high-frequency customers who know exactly what they want — the current flow requires the same steps regardless of familiarity.

🟠 Philz Coffee

Recent order shortcuts address efficiency for returning users. Customization has a ceiling: special requests require free-text entry rather than a structured modifier system, placing more cognitive burden on the user than necessary.

H08

Aesthetic and Minimalist Design

☕ Starbucks

The interface is clean and visually polished. The main tension is the home screen — promotional banners and featured articles compete with the primary ordering task, adding clutter at the entry point of every session.

🟠 Philz Coffee

Prioritizes a clean, focused layout with large, touch-friendly buttons in plain language. Each screen has a clear primary action, and supplementary content doesn't compete for the user's attention.

H09

Help Users Recognize, Diagnose, and Recover from Errors

☕ Starbucks

Error dialogs are clear and actionable — written in plain language, never technical jargon, and always paired with a specific next step. Users can resolve most errors without leaving the flow or restarting.

"Skip to menu?" — offers a choice, not an error dump.

Blocks cart gracefully with an immediate resolution path.

🟠 Philz Coffee

Error handling is direct and immediate. Invalid payment information is flagged inline the moment it's entered, guiding users to the specific field to correct — no submission required, no vague message. The error names the problem and points to the solution.

H10

Help and Documentation

☕ Starbucks

A help section exists with searchable FAQs, but it requires navigation to find — it's not surfaced where users are most likely to need it. Making live support options more prominent within the ordering flow would reduce friction for users who get stuck.

🟠 Philz Coffee

The app is largely self-explanatory — a testament to clear design. The gap is the absence of any easy-access help section. For users who need assistance, there's no clear path to support, which works against the approachable personality the app otherwise projects.

05 // VERDICT

What the data says

Starbucks wins on consistency, error prevention, and recovery — heuristics where scale and iteration provide a natural advantage. Philz wins where those advantages don't apply: real-world language, system feedback, and minimalist design. The heuristics Philz clears are the ones that require deliberate design intent, not budget.

☕ Starbucks — Recommendation

Add post-order cancellation during the preparation window. This single addition closes the largest gap in User Control and Freedom without requiring a redesign — and it's the one moment users most need an exit.

🟠 Philz Coffee — Recommendation

Keep the user on the product detail page after adding an item to the cart, letting them decide when they're finished rather than auto-navigating away. Pair this with a help section in the main nav to close both their control and documentation gaps.

06 // REFLECTION

Method Note

Heuristic evaluation surfaces what's there — it doesn't tell you whether users actually notice, recover from, or care about the issues found. Several of Starbucks' gaps (buried help, no post-order cancellation) likely go unnoticed by most users because the primary flow is so polished. Philz's wins on language and feedback may disproportionately benefit users who are new to the brand. This evaluation is the beginning of the question, not the end — the right next step for either finding would be a moderated usability test on the specific friction points identified.
