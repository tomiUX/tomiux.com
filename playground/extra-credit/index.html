№ 001
The Runaway X
CLOSE BUTTON
// The Problem

Close buttons that are too small, too close to ads, or positioned to maximize accidental clicks. The dark pattern: make dismissing hard so users give up and comply.

// The Pattern

A button that moves when you try to click it. Frustrating? Yes. Familiar? Absolutely. You've seen this on mobile pop-ups, cookie banners, and subscription prompts.

// The Fix

Close buttons should be large (minimum 44×44px touch target), clearly labeled, and static. Dismissing is a valid user action and deserves to be easy.

№ 002
Eternal Spinner
LOADER
// The Problem

Loading spinners that never resolve — or give no indication of how long the wait will be. Users don't know if it's working, broken, or waiting for them.

// The Pattern

A progress bar that climbs confidently to 79%... and stops. Familiar from countless software installs, uploads, and checkouts. Time passes. The bar stays at 79%.

// The Fix

Show real progress where possible. If indeterminate, communicate that honestly. Always provide an estimated time or a way out. Silence during waiting is a UX failure.

№ 003
The Un-Undoable
DESTRUCTIVE ACTION
// The Problem

Destructive actions — delete, archive, send — with no confirmation and no undo. One misclick and it's gone. The user has no recovery path.

// Two Demos

Delete items on the left — no undo. Delete on the right — an undo toast appears for 5 seconds. Feel the difference between a system that trusts you and one that catches you.

// The Fix

Provide a brief undo window for all destructive actions. Gmail's "Undo Send" is the gold standard. The action completes immediately (satisfying) but is reversible (safe).

№ 004
Consent Trap
DARK PATTERN
// The Problem

Cookie banners and permission forms designed to obscure opt-outs, pre-check boxes users didn't choose, and bury dissent in legalese. Consent shouldn't require a law degree.

// The Pattern

Three checkboxes. One is the thing you want. Two are things you didn't ask for, pre-checked, in font sized to discourage reading. Submit and find out what you agreed to.

// The Fix

All non-essential consents should default to unchecked. Language should be plain. Declining should be as easy as accepting. This is also a legal requirement under GDPR and CCPA.
